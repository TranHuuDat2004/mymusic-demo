Giới Thiệu MyMusic Player

MyMusic Player là một giao diện trình phát nhạc trực tuyến được thiết kế và phát triển nhằm mang đến trải nghiệm nghe nhạc đơn giản, hiện đại và trực quan, lấy cảm hứng từ các nền tảng phổ biến như Spotify và Zing MP3.

Mục Tiêu Dự Án

Mục tiêu chính của dự án là xây dựng một minh chứng (proof-of-concept) về khả năng phát triển giao diện người dùng (UI) và trải nghiệm người dùng (UX) cho ứng dụng web, tập trung vào các yếu tố sau:

  • Thiết kế Giao diện: Tạo lập một giao diện thẩm mỹ, gọn gàng và dễ dàng điều hướng cho người sử dụng.
  • Chức năng Cốt lõi: Triển khai các chức năng cơ bản của một trình phát nhạc, bao gồm phát/tạm dừng, hiển thị tiến trình và điều chỉnh âm lượng.
  • Tổ chức Nội dung: Trình bày hiệu quả các danh sách bài hát và thông tin liên quan.
  • Thiết kế Đáp ứng (Responsive): Đảm bảo giao diện hiển thị và hoạt động tối ưu trên các kích thước màn hình khác nhau, từ máy tính để bàn đến thiết bị di động.
  • Tương tác Động: Tận dụng JavaScript để tạo ra các yếu tố giao diện động và phản hồi tương tác người dùng một cách mượt mà.
  • Điều Khiển Nâng Cao: Cung cấp các tùy chọn điều khiển phát nhạc quen thuộc như phát ngẫu nhiên (shuffle) và các chế độ lặp lại (repeat all, repeat one).
  • Phản Hồi Người Dùng: Sử dụng hệ thống thông báo (notifications) để xác nhận các hành động quan trọng, cải thiện trải nghiệm tương tác.

Dự án cũng là cơ hội để thực hành và củng cố các kỹ năng phát triển front-end cốt lõi.

Nền Tảng Công Nghệ

MyMusic Player được xây dựng hoàn toàn bằng các công nghệ front-end cơ bản:

  • HTML5: Cung cấp cấu trúc ngữ nghĩa cho nội dung trang web.
  • CSS3: Định dạng giao diện, bao gồm các kỹ thuật layout hiện đại như Flexbox, Grid Layout và sử dụng biến CSS (Custom Properties) để quản lý theme.
  • JavaScript (ES6+): Xử lý logic ứng dụng, tương tác người dùng, quản lý trạng thái trình phát và cập nhật giao diện động (không sử dụng framework/thư viện bên ngoài).
  • Hosting: Triển khai dưới dạng trang web tĩnh thông qua GitHub Pages.

Phạm Vi và Giới Hạn

MyMusic Player là một ứng dụng web tĩnh (static web application), được triển khai trên GitHub Pages. Do đó, ứng dụng hoạt động hoàn toàn phía client (trình duyệt) và có các giới hạn sau:

  • Dữ liệu tĩnh: Danh sách bài hát, thông tin nghệ sĩ và ảnh bìa được định nghĩa cố định trong mã nguồn (JavaScript) và không thể được người dùng thêm/sửa/xóa thông qua giao diện.
  • Không có tính năng phía máy chủ: Không có cơ sở dữ liệu, chức năng đăng nhập/đăng ký người dùng, lưu trữ playlist cá nhân trên server, hoặc khả năng tải lên tập tin nhạc.
  • Hoạt động độc lập: Không có sự đồng bộ hóa trạng thái nghe nhạc giữa các thiết bị hoặc phiên làm việc khác nhau.

Các giới hạn này phản ánh bản chất của một dự án front-end tĩnh tập trung vào giao diện và trải nghiệm người dùng phía client.

About MyMusic Player

MyMusic Player is an online music player interface designed and developed to provide a simple, modern, and intuitive music listening experience, inspired by popular platforms like Spotify and Zing MP3.

Project Goals

The main goal of the project is to build a proof-of-concept demonstrating capabilities in developing user interface (UI) and user experience (UX) for web applications, focusing on the following aspects:

  • Interface Design: Create an aesthetically pleasing, clean, and easy-to-navigate interface for users.
  • Core Functionality: Implement basic music player functions, including play/pause, progress display, and volume control.
  • Content Organization: Effectively present song lists and related information.
  • Responsive Design: Ensure the interface displays and functions optimally across various screen sizes, from desktops to mobile devices.
  • Dynamic Interaction: Utilize JavaScript to create dynamic interface elements and respond smoothly to user interactions.
  • Advanced Controls: Provide familiar playback control options such as shuffle and various repeat modes (repeat all, repeat one).
  • User Feedback: Utilize a notification system to confirm important actions, enhancing the interactive experience.

The project also serves as an opportunity to practice and reinforce core front-end development skills.

Technology Stack

MyMusic Player is built entirely with fundamental front-end technologies:

  • HTML5: Provides the semantic structure for the web page content.
  • CSS3: Styles the interface, including modern layout techniques like Flexbox, Grid Layout, and using CSS Custom Properties for theme management.
  • JavaScript (ES6+): Handles application logic, user interactions, player state management, and dynamic interface updates (without using external frameworks/libraries).
  • Hosting: Deployed as a static website via GitHub Pages.

Scope and Limitations

MyMusic Player is a static web application, deployed on GitHub Pages. Therefore, the application operates entirely on the client-side (browser) and has the following limitations:

  • Static Data: Song lists, artist information, and album art are hardcoded in the source (JavaScript) and cannot be added/edited/deleted by users through the interface.
  • No Server-Side Features: There is no database, user login/registration functionality, server-side personal playlist storage, or music file upload capability.
  • Standalone Operation: There is no synchronization of listening status between different devices or sessions.

These limitations reflect the nature of a static front-end project focused on client-side interface and user experience.
